Feature Groups

The MVP is intentionally scoped around the smallest complete SaaS workflow: sign in, enter a client workspace, ask coding questions, retrieve trusted context, cite sources, track usage, and operate the platform safely.

User Access

Login, tenant workspace, and role boundaries

  • User signup/login through email/password or OAuth.
  • Each client company gets its own workspace.
  • Developer, client admin, and central SaaS admin roles.
Developer Chat

Source-backed coding assistant

  • Streaming AI chat answers.
  • Code-aware Markdown with syntax-highlighted examples.
  • Python, PHP, and .NET support through prompts, parsing, and RAG evaluation.
Knowledge Layer

Universal plus tenant-specific RAG

  • RAG ingestion for documents and repository files.
  • Shared universal coding knowledge collection.
  • Tenant-specific private chunks retrieved with strict filters.
Evidence

Citations and answer trust

  • Show where each answer came from.
  • Support source-backed debugging and onboarding.
  • Reduce hallucination risk through source-first response patterns.
Usage

Client and user-wise metering

  • Track input and output tokens.
  • Client admins see tenant and individual user-wise usage.
  • Developers see only personal usage.
Operations

Provider monitoring and billing readiness

  • Central SaaS admin console for separate client monitoring.
  • Billing-ready usage records for future Stripe integration.
  • Logs, monitoring, and guardrails for multi-tenant operation.

SaaS Workflow

1. Sign inUser enters the correct client workspace through tenant membership.
2. UploadClient admin uploads docs, repositories, standards, and runbooks.
3. RetrieveQdrant returns universal coding knowledge plus tenant-specific private chunks.
4. AnswerHosted LLM generates cited answers for Python, PHP, .NET, and DevOps questions.
5. MeterUsage is tracked by tenant and user for packages, support, and billing.

Explicitly Out of MVP

  • Training a 70B or 79B model.
  • Complex fine-tuning.
  • Enterprise SSO or air-gapped deployment.
  • Full IDE extension or full pull request automation.
  • Unlimited external MCP tools.
